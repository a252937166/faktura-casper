# Security Policy

## Supported versions

The `master` branch and the latest release tag receive fixes.

## Reporting a vulnerability

Please use
[GitHub private vulnerability reporting](../../security/advisories/new)
for anything sensitive — do not open a public issue for exploitable bugs.
Reports are triaged within a few days.

Scope notes: this is a testnet hackathon project. Keys committed under
`keys/` are throwaway Casper Testnet keys used by the demo personas; they
hold no mainnet value. Do not reuse them.
